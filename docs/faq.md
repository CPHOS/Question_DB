# FAQ

## 为什么数据库里既保存路径，也保存 LaTeX 文本？
路径和文本承担的职责不同。路径用于追溯原始工程、重新编译和回写；LaTeX 文本快照用于查询、导出、比对、审计和跨环境迁移。当前题量不大，因此双存的成本很低，但能显著降低数据库对外部目录结构的依赖。

## 为什么既要保存 xlsx，又要保存 question_stats？
xlsx 是原始统计工作簿，保真且可追溯；`question_stats` 是便于检索、筛选和后续难度分析的结构化结果，两者用途不同。

## PDF 还有用吗？
有，但不再作为主索引。PDF 更适合作为历史来源或发布产物，主数据仍然以 LaTeX 文件和卷内题目索引为准。

## 现在的 question_id 和 paper_index 怎么分工？
`question_id` 是全局唯一标识，`paper_index` 是卷内顺序。后者取代旧设计里依赖页码定位的做法。

## 当前内部 API 的导入方式是什么？
当前版本采用“服务器路径导入”。也就是说，LaTeX 文件、图片和 xlsx 工作簿先放到服务器目录中，再通过内部 API 提交 bundle 路径、workbook 路径或统计 CSV 路径，由服务读取文件并写入数据库。这样可以避免通过 HTTP 直接上传大量文件，同时保留清晰的来源路径。
